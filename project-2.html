Meal Planner

Solving my own pain while learning the Django web framework.

Project Overview

It's a grinding everyday task: producing a healthy, enjoyable and varied meal every day for ourselves and our kids.

As soon as I was able, I created an online meal-scheduling tool that would speed up the thinking process and help us log, manage and get inspiration for the rotation of our family dinners.

First I built it as a basic calendar in vanilla JavaScript. It used my newfound CSS flexbox and Grid skills for the frontend, and local storage to hold the library of dishes and list of meals created for specific dates.

The next version of the planner was a full rewrite, because I used it as an excellent way to improve my understanding of the Django web framework, practise modelling data and building out a traditional server-side CRUD app.

The core behaviour is straightforward: storing a library of Dish objects (a title and description) and the concept of a Meal object (a foreign key to a particular Dish, on a particular date, plus a notes field). A key aim, however, was for the app to be able to suggest dishes that we hadn't cooked for some time, as it's so easy to fall into a rut of repetitively cooking the same meals. With this in mind, the core features became:

  • A quick way to view the current week's scheduled meals, as well as the previous and next week's plan — this involves datetime-based database queries to get the right "window" of data.
  • A dynamically generated list of suggested dishes for inspiration — based on a simple algorithm that seeks the least recently selected dishes.
  • A shortcut for adding a new meal from the main dish list - the "Plan It!" button. This special link uses a dish-specific querystring to pre-populate the form in the "add a meal" page, so the user doesn't have to rummage around for it.

My focus was on utility over style, so I pulled in basic styling using the Simple CSS framework, which let me keep my mind on the data.
